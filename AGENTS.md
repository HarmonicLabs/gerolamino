# Agents

Instructions for AI agents working on the Gerolamino codebase.

## General Rules

- Read existing code before modifying it. Understand the patterns in use.
- Never use `as Type` typecasts. Only `as const`. Use Effect pipelines/Schema.
- Use `Schema.TaggedClass` for domain types, `Schema.TaggedErrorClass` for errors.
- Use `Effect.gen` with `yield*`, not nested `Effect.flatMap`.
- Use `Config.string()` / `Config.number()` / `Config.duration()` for env vars
  (NOT `Config.int` — that doesn't exist in v4), not `process.env`.
- Use `Effect.log*` (NOT `console.*`) for logging. `console.*` is forbidden
  in `packages/*/src/**` and `apps/*/src/**` outside bench harnesses.
- All imports at top of file. No lodash. No dynamic `import()` inside functions.
- Don't write nested-pipe chains like `x.pipe(a).pipe(b)`; coalesce to
  `x.pipe(a, b)`.
- Run tests with `bunx --bun vitest`, not `bun test` or `jest`. Never
  `Effect.runPromise` in test bodies — use `@effect/vitest` `it.effect` /
  `it.layer` / `it.prop`.
- Run builds with `bunx --bun tsgo --build`. Never invoke stock `tsc`.
- Commit `bun.nix` when `bun.lock` changes (auto-generated by postinstall).
- XState is retained ONLY for `packages/storage/src/machines/chaindb.ts`
  (genuine parallel-region). All other machines have been migrated to
  `Stream` / `Effect.gen` / `SubscriptionRef`.
- Check `~/code/reference/effect-smol/` for Effect v4 source before guessing
  at an API. Many plan-era API claims are stale against the current beta;
  always verify.

## Effect-TS v4 Beta

This project uses Effect v4 beta. Key API differences from v3 and common
wave-2 corrections (verified 2026-04-22 against `~/code/reference/effect-smol/`):

- `Schema.TaggedErrorClass` (not `Data.TaggedError`)
- `Effect.catch` (not `Effect.catchAll`)
- `Config.string()` / `Config.number()` / `Config.duration()` for env config
- `Schema.Class` for data types with methods
- `yield*` inside `Effect.gen` (not `_` adapter)
- `Effect.forkChild` / `Effect.forkScoped` (NOT `Effect.fork` — doesn't exist)
- `Effect.exit` + `Exit.isFailure` (NOT `Effect.either`)
- `Stream.runCollect` returns `Effect<Array<A>, E, R>` directly (no `Chunk.toArray`)
- `Atom.debounce(source, "16 millis")` takes `Duration.Input` directly
  (NOT `{ delay }` options object). `Atom.batchSet` doesn't exist.
- `Schedule.exponential(1000, 1.2).pipe(Schedule.jittered, Schedule.both(...))`
  — `both` IS the intersection combinator; `Schedule.intersect` doesn't exist.
- `RateLimiter` lives at `effect/unstable/persistence` (not `@effect/experimental`).
- `RpcSerialization.of({ makeUnsafe, contentType, includesFraming })` —
  factory key is `makeUnsafe` (NOT `unsafeMake`), and `includesFraming` is required.
- `RpcClient.layerProtocolWorker({ size, concurrency })` — `spawn` is NOT an
  option; provide via `Worker.layerSpawner(fn)` as a separate layer.
- `RpcGroup.make(...).toLayer(handlers)` IS correct (pair with
  `RpcServer.layer(group, options)`; there's no `RpcServer.serve`).
- `HttpApi` lives at `effect/unstable/httpapi/` (lowercase, NOT `unstable/http/`).
- `PubSub.unbounded` uses a DroppingStrategy — use `bounded(n)` if you need
  deterministic pre-subscription delivery semantics.
- For PubSub test sync: follow the `effect/test/PubSub.test.ts` canonical
  pattern — `Latch.make()` + `PubSub.subscribe(p)` inside `Effect.scoped` +
  `Effect.forkChild({ startImmediately: true })`.

## Cross-Package Imports

Use tsconfig.base.json path aliases:

```typescript
import { decodeMultiEraBlock } from "ledger/lib/block/block.ts";
import { CborValue } from "codecs";
```

Do NOT use relative paths like `../../packages/ledger/src/...`.

## WASM Packages

wasm-plexer and wasm-utils are Rust crates built via Nix. Do not attempt
to build them with `cargo` or `wasm-pack` directly in the dev shell -
use `nix build .#wasm-plexer` or `nix build .#wasm-utils`.

TypeScript packages import WASM bindings from:

- `wasm-plexer` -> `packages/wasm-plexer/result/`
- `wasm-utils` -> `packages/wasm-utils/pkg/`

## Nix Build System

- `nix build .#ts-packages` builds all TS libraries
- `nix build .#bootstrap-image` builds the OCI container
- `nix flake check --allow-import-from-derivation` validates everything
  including deploy-rs + treefmt
- WASM outputs are injected into the TS source tree via `ln -sfn` in
  `postUnpack` (NOT `cp -r` — plan Phase 0a fix)
- bun2nix uses `--backend=copyfile` (hardlinks fail in Nix sandbox)
- Cachix substituter `harmoniclabs.cachix.org` is the project cache.
  CI wires via `cachix/cachix-action@v15`.

## Testing

- Unit tests: `bunx --bun vitest run packages/<name>`
- Integration tests against a live preprod relay: `VITE_INTEGRATION=1 bunx --bun vitest run packages/miniprotocols`
- Mithril V2LSM fixture tests: `MITHRIL_FIXTURE_ENABLED=true bunx --bun vitest run apps/bootstrap`
- Benchmarks: `bunx --bun vitest bench packages/<name>`
- `@effect/vitest` test-harness required — raw `vitest` `it` allowed only
  in pure-schema round-trip tests with no Effect involvement.

## Deployment

- Production NixOS config: `nix/machine-configs/production.nix`
- Deploy: `nix run .#deploy` (deploy-rs with magic rollback; confirmTimeout 300s)
- Target: `decentralizationmaxi.io:2222` (SSH port 2222, key-only auth)
- Prod services: `cardano-node` (preprod, V2LSM, master-tracked) +
  self-hosted `mithril-aggregator` + `mithril-signer` (see
  `nix/machine-configs/mithril-services.nix`)
- Hardware migration via `nixos-anywhere`: see `docs/deployment.md` for
  the kexec → disko → install → zfs-send → deploy-rs sequence.
- CI: GitHub Actions on `ubuntu-latest` with DeterminateSystems/determinate-nix-action.
